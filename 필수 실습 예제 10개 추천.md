# 필수 실습 예제 10개 추천

작성일: 2026-06-14

## 목적

v1.5 플레이북에는 프롬프트 예제가 43개 들어 있다. 하지만 2시간 실습 교육에서 43개를 모두 진행하기에는 시간이 부족하다.

이 문서는 지원팀 대상 Codex 실습 교육에서 꼭 포함하면 좋은 실습 예제 10개를 추천한다. 기준은 다음과 같다.

- 초보자가 Codex의 역할을 빠르게 이해할 수 있는가
- 교육생 PC 환경이 달라도 비교적 공통으로 따라 할 수 있는가
- 실제 업무 자동화 감각을 느낄 수 있는가
- 개인정보, 비밀번호, 사내 시스템 권한 문제를 과하게 건드리지 않는가
- 2시간 안에 설명, 복붙, 결과 확인, 짧은 회고까지 가능한가

## 추천 실습 10개 상세

### 1. #1 Codex와 ChatGPT 차이 설명

추천 이유:
처음부터 파일 수정이나 자동화로 들어가면 초보자는 부담을 느낄 수 있다. 먼저 ChatGPT와 Codex의 차이를 업무 예시로 이해하면, 이후 실습에서 왜 Codex 앱을 쓰는지 자연스럽게 받아들일 수 있다.

진행 포인트:
- ChatGPT는 대화와 정리에 강하고, Codex는 프로젝트 폴더와 파일 작업에 강하다는 차이를 잡아준다.
- "Codex는 내 PC의 작업 폴더를 보고 일을 도와주는 동료" 정도로 설명하면 이해가 빠르다.

주의:
도구 설명이 너무 길어지면 실습 시간이 줄어드니 5분 안에 끝낸다.

### 2. #5 이 폴더 뭐 하는 프로젝트야?

추천 이유:
Codex의 핵심 체감 포인트다. 교육생이 플레이북 폴더를 열고 "이 폴더가 뭐 하는 프로젝트인지 설명해줘"라고 요청하면, Codex가 실제 파일 구조를 읽고 설명한다. 이 경험이 있어야 이후 파일 정리, 데이터 분석, 스킬 만들기가 자연스럽다.

진행 포인트:
- 반드시 `v1.5` 폴더 또는 배포받은 플레이북 폴더를 프로젝트로 열게 한다.
- 결과에서 `index.html`, `assets`, `docs`, `examples`가 어떻게 설명되는지 확인한다.

주의:
다른 폴더를 열면 결과가 달라진다. 실습 전 "현재 Codex에 어떤 폴더가 열려 있는지" 확인하게 한다.

### 3. #9 + #10 대충 요청 vs 구조화 요청 비교

추천 이유:
프롬프트 기본기의 핵심을 가장 짧은 시간에 보여준다. 같은 주제라도 대충 묻는 것과 목표, 맥락, 제약, 완료 조건을 넣는 것의 결과 차이가 눈에 보인다.

진행 포인트:
- 먼저 #9를 실행해서 답변이 넓고 추상적으로 나오는 것을 확인한다.
- 이어서 #10을 실행해서 단계, 조건, 안전장치가 훨씬 구체적으로 나오는 것을 비교한다.
- "좋은 프롬프트는 긴 프롬프트가 아니라 일의 기준이 들어간 프롬프트"라고 정리한다.

주의:
두 개를 한 세트로 진행한다. 둘 중 하나만 하면 비교 효과가 약하다.

### 4. #12 Codex에게 프롬프트부터 만들어 달라고 하기

추천 이유:
초보자는 "무엇을 어떻게 시켜야 할지 모르겠다"에서 막힌다. 이 실습은 그럴 때 Codex에게 먼저 프롬프트를 만들어 달라고 할 수 있다는 실용적인 방법을 알려준다.

진행 포인트:
- 교육생이 자기 업무 하나를 떠올려 `[하고 싶은 일]`에 바꿔 넣어보게 한다.
- Codex가 만들어준 프롬프트를 다시 Codex에 넣는 흐름을 설명한다.

주의:
실제 고객정보, 계정, 내부 URL은 넣지 않도록 안내한다.

### 5. #13 외부 AI 사용 전 가장 먼저 확인할 것

추천 이유:
AI 실습에서 가장 먼저 잡아야 할 기준은 개인정보와 비밀값 보호다. 이 내용을 초반에 넣으면 뒤의 `.env`, 이메일, 사내 시스템 자동화 실습을 더 안전하게 진행할 수 있다.

진행 포인트:
- API key, 비밀번호, OTP, 고객정보, 내부 URL은 직접 넣지 않는다는 기준을 강조한다.
- "편리함보다 정보 보호가 먼저"라는 한 줄 결론을 교육생이 기억하게 한다.

주의:
보안 설명이 겁주기로만 들리지 않도록, 마스킹과 `.env` 같은 대체 방법을 함께 설명한다.

### 6. #15 .env에 Knox 메일 설정하기

추천 이유:
실제 업무 자동화에서는 계정, 비밀번호, 서버 주소를 코드나 프롬프트에 직접 넣으면 위험하다. `.env` 개념을 잡아두면 뒤의 POP3/SMTP, 사내 시스템 자동화, 스킬 배포까지 연결된다.

진행 포인트:
- 실습에서는 실제 비밀번호를 넣지 않고 빈 값 또는 `<SECRET>`로 둔다.
- `.env`와 `.env.example`의 차이를 설명한다.
- 공유 전 `.env`가 포함되지 않았는지 확인하는 습관을 강조한다.

주의:
실제 Knox 비밀번호나 AD 비밀번호를 화면에 입력하지 않는다.

### 7. #24 파일 종류별로 분류하기

추천 이유:
파일 자동 정리는 Codex 초보자가 빠르게 효과를 느끼기 좋은 실습이다. 특히 이 프롬프트는 "아직 파일은 이동하지 마"라고 되어 있어 안전하다.

진행 포인트:
- 먼저 계획만 받는다.
- 결과에서 문서, 엑셀, 이미지, 압축파일처럼 분류 기준이 잘 나오는지 본다.
- 실제 이동/삭제는 사람 확인 후에만 진행해야 한다고 설명한다.

주의:
파일 삭제나 이동까지 바로 시키지 않는다.

### 8. #28 현재 환경의 도구/스킬 확인하기

추천 이유:
Codex가 그냥 답변만 하는 챗봇이 아니라, 도구와 스킬을 통해 파일, 브라우저, 문서, 엑셀 등을 다룰 수 있다는 점을 이해하게 해준다.

진행 포인트:
- 교육생 PC마다 사용 가능한 도구/스킬이 다를 수 있다는 점을 설명한다.
- "Codex에게 일을 맡기기 전에 현재 사용할 수 있는 도구를 확인할 수 있다"는 습관을 만든다.

주의:
결과가 강사 PC와 다르게 나와도 오류가 아니라 환경 차이라는 점을 알려준다.

### 9. #36 PSI Data 분석 계획 세우기

추천 이유:
실제 업무에서 가장 많이 쓰일 가능성이 높은 유형이다. 데이터를 바로 분석하게 하기보다 먼저 분석 계획을 세우게 하므로 안전하고, 업무 적용성이 크다.

진행 포인트:
- `examples/psi-dashboard/PSI-sample.xlsx` 파일이 있는지 먼저 확인한다.
- Codex가 필요한 컬럼, 품질 점검 항목, 분석 단계, 표/그래프 후보를 어떻게 제안하는지 본다.
- 분석 전에 계획을 세우는 이유를 설명한다.

주의:
실제 고객명, 거래처명, 내부 코드가 있는 파일은 그대로 올리거나 공유하지 않는다.

### 10. #43 내일부터 쓸 개인 실행 계획 만들기

추천 이유:
교육 마지막에는 "재미있었다"보다 "내일부터 뭘 해볼지"가 남아야 한다. 이 프롬프트는 오늘 배운 내용을 자기 업무에 적용하는 계획으로 바꿔준다.

진행 포인트:
- 교육생 각자가 자기 업무 기준으로 답변을 받게 한다.
- 바로 해볼 3가지, 넣으면 안 되는 정보, 반복 업무 후보를 확인한다.
- 결과를 개인 메모로 남기게 하면 교육 효과가 오래 간다.

주의:
회사 정책상 외부 AI에 입력하면 안 되는 업무 정보는 마스킹해서 적게 한다.

## 최종 추천

필수 10개는 설명형, 비교형, 보안형, 파일 작업형, 데이터 분석형, 마무리형이 골고루 섞여 있다. 2시간 교육에서는 이 10개를 중심으로 진행하고, POP3/SMTP/GSCM/Agent 자동화는 권한과 준비물이 되는 사람에게 선택 실습으로 보여주는 구성이 가장 안정적이다.
