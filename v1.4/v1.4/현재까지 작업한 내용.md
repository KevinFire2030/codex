### 2. `04. 프롬프트 기본기` 섹션 개선

공식 문서 기준 흐름으로 정리했습니다.

현재 흐름:

1. 목표 / 맥락 / 제약 / 완료 조건
2. 공식 문서로 프롬프트 작성법 확인하기
3. 대충 요청 vs 구조화해서 요청 비교
4. 막히면 Codex에게 프롬프트부터 만들어 달라고 하기

추가된 실습:

- Codex 공식 문서에 프롬프트 작성 가이드가 있는지 확인하는 프롬프트
- RAG 챗봇 주제로 대충 쓴 프롬프트와 구조화한 프롬프트 비교
- Codex에게 먼저 좋은 프롬프트를 만들어 달라고 하는 메타 프롬프팅 실습

RAG 비교 실습 주제:

```text
지원팀 경비 정산 매뉴얼 PDF 기반 RAG 챗봇
```

핵심 메시지:

- 대충 요청하면 일반론이 나온다.
- 목표, 맥락, 제약, 완료 조건을 넣으면 파일 구조, 보안 기준, 완료 기준까지 구체화된다.
- 프롬프트를 잘 못 쓰겠으면 Codex에게 프롬프트를 먼저 만들어 달라고 할 수 있다.

## 검증한 내용

각 변경 후 주로 아래를 확인했습니다.

```powershell
node --check assets\js\app.js
```

그리고 HTML 내부의 로컬 `href` / `src` 대상 파일이 존재하는지도 확인했습니다.

브라우저 자동 검증은 `file://` 로컬 파일 접근이 in-app Browser 정책상 차단되어 진행하지 못했습니다. 회사 PC에서는 파일 탐색기에서 `index.html`을 직접 열어 화면 여백과 복사 버튼 동작을 한 번 확인하면 좋습니다.
