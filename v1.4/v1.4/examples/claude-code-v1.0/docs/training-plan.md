# Training Plan v1.0

## 교육 대상

지원팀 구성원

## 진행 방식

참석자는 개인 PC에서 왼쪽에 `examples/claude-code-v1.0/index.html` 플레이북을 열고, 오른쪽에 터미널이나 VS Code의 Claude Code 화면을 실행합니다. 각 실습에서는 플레이북의 예시 프롬프트를 복사해 Claude Code에 붙여 넣고 결과를 확인합니다.

## 2시간 진행표

| 시간 | 주제 | 실습 |
| --- | --- | --- |
| 0:00-0:10 | 교육 목표와 보안 원칙 | 민감정보 입력 금지, 명령 실행 전 검토 |
| 0:10-0:20 | 처음 만나는 Claude Code | Claude와 Claude Code 차이 이해 |
| 0:20-0:30 | 실행 환경 선택 | 터미널, VS Code, 데스크톱/웹 사용 방식 비교 |
| 0:30-0:45 | 첫 작업 시키기 | "이 폴더 뭐 하는 프로젝트야?" |
| 0:45-1:00 | 확장 실습 | Codex Playbook을 Claude Code Playbook으로 바꾸는 계획 세우기 |
| 1:00-1:10 | 프롬프트 기본기 | 목표, 맥락, 제약, 완료 조건 |
| 1:10-1:20 | 개인정보 환경변수 설정 | .env, 계정, URL, 작업 폴더 분리 |
| 1:20-1:30 | 파일/문서 작업 | 원본 보존, 결과 파일 생성, 변경사항 검토 |
| 1:30-1:40 | 사내 시스템 UI 자동화 | 권한, 감사 로그, 승인 필요 작업 구분 |
| 1:40-1:50 | Skills, slash commands, MCP | 반복 업무를 재사용 가능한 절차로 정리 |
| 1:50-2:00 | 데이터/엑셀/보안 | PSI 대시보드, 엑셀 병합, DRM |

## 핵심 메시지

- Claude Code는 단순히 답변하는 챗봇이 아니라, 현재 프로젝트 폴더를 읽고 파일 수정과 명령 실행을 도와주는 에이전트형 작업 도구입니다.
- 처음에는 파일을 수정시키지 말고 읽기 전용으로 구조와 계획을 설명하게 합니다.
- Claude Code가 제안하는 파일 변경과 명령은 사람이 검토하고 승인합니다.
- 원본 파일은 보존하고 결과 파일은 별도 위치에 생성합니다.
- 반복 업무는 프롬프트, slash command, skill, MCP 같은 형태로 정리할 수 있습니다.
- 보안, 권한, DRM 정책은 실습 결과보다 먼저 확인합니다.
