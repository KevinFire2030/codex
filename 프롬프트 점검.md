# 프롬프트 점검

점검일: 2026-06-14

점검 대상: `v1.5/index.html`의 프롬프트 모음 #1 ~ #43

점검 목적: 이 플레이북은 지원팀 대상 Codex 실습 교육에서 사용한다. 교육 진행자는 컴퓨터 화면 왼쪽에 플레이북을 띄우고, 오른쪽에 Codex 앱을 실행한다. 교육생은 설명을 들은 뒤 실습 예시 프롬프트를 복사해서 Codex 앱에 붙여 넣고 결과를 확인한다.

점검 기준:
- 교육생 PC 환경이 달라도 공통으로 이해하거나 실행할 수 있는가
- 내 PC 기준 경로, 개인 계정, 사내 권한, 로그인 세션에 과하게 의존하지 않는가
- 실제 메일 발송, 파일 삭제, 민감정보 노출 같은 위험 동작이 바로 실행되지 않는가
- 실습 예제로서 결과가 명확하고 초보자가 차이를 느끼기 좋은가
- 수정이 필요해 보이는 경우에도 이번 요청에 따라 실제 수정/푸쉬는 하지 않고 개선 제안만 남긴다

주의: 아래 결과는 실제 사내 시스템 접속, 실제 POP3/SMTP 로그인, 실제 메일 발송, 실제 DRM 문서 열람을 수행한 결과가 아니다. 교육생 공통 PC에서 복붙했을 때의 예상 실행 흐름을 기준으로 한 모의 실행 및 품질 점검 결과다.

## 종합 결론

전체 43개 중 대부분은 교육용 설명, 계획 수립, 안전 점검, 샘플 파일 분석 중심이라 공통 실습에 적합하다.

다만 다음 프롬프트는 교육생 PC마다 결과가 달라질 수 있으므로 강사가 사전 안내를 붙이는 것이 좋다.

- #15, #16, #17, #20, #21, #23, #30, #31: 사내 계정, 사내망, 권한, 로그인된 Chrome 세션에 의존한다.
- #7, #31, #34, #37, #39, #41: 파일 생성/설치/결과물 생성으로 이어질 수 있어 실습 전 작업 폴더를 명확히 해야 한다.
- #18: "방금 SMTP로 보낸 메일"이라는 직전 상황 의존성이 있어, SMTP 권한이 없는 교육생에게는 가정형 예시로 설명하는 편이 안전하다.
- #33, #34: 스킬 ZIP 다운로드와 설치가 선행되어야 자연스럽다.
- #37: 프롬프트 안의 `PSI.xlsx`가 실제 샘플 파일명과 다를 수 있어 `examples/psi-dashboard/PSI-sample.xlsx` 기준으로 바꾸면 더 공통적이다.

이번 요청에서는 수정하지 않았고, 아래에 수정 제안만 기록했다.

## 수정 필요 후보 정리

이번 요청에서는 실제 수정하지 않았다. 다음에 수정한다면 우선순위는 아래와 같다.

1. #37 `PSI.xlsx`를 `examples/psi-dashboard/PSI-sample.xlsx`로 바꾸기
   - 이유: 배포본에 포함된 샘플 파일 기준으로 맞추면 교육생 PC 공통 실행 가능성이 높아진다.

2. #16 POP3 실습에 권한 없는 경우의 대체 출력 추가
   - 이유: POP3 권한이 없는 교육생도 같은 흐름을 배울 수 있어야 한다.
   - 예: "접속 권한이 없으면 실제 접속하지 말고 필요한 설정값과 예상 결과 형식을 설명해줘."

3. #17 SMTP 실습의 실제 발송 금지 문구 강화
   - 이유: 발송 권한이 있더라도 실습 중 실수로 메일이 나가면 안 된다.
   - 예: "코드 작성이나 실제 발송은 하지 말고 DRY_RUN 미리보기만 작성해줘."

4. #18 한글 깨짐 실습을 가정형으로 변경
   - 이유: SMTP 실습을 실제로 하지 않은 교육생도 자연스럽게 실행할 수 있다.
   - 예: "SMTP로 보낸 메일이 아래처럼 깨졌다고 가정해줘."

5. #20, #21 사내 시스템 UI 자동화 프롬프트에 실패 시 대체 답변 조건 추가
   - 이유: 로그인/권한/사내망 상태가 교육생마다 다르다.
   - 예: "접속할 수 없으면 실제 실행하지 말고 필요한 권한, 예상 절차, 확인 체크리스트를 알려줘."

6. #23 녹화 폴더 존재 확인 단계 추가
   - 이유: `%USERNAME%`은 공통이지만 녹화 폴더명은 매번 다르다.
   - 예: "먼저 녹화 폴더가 존재하는지 확인하고, 없으면 필요한 파일 목록을 알려줘."

7. #31 스킬 생성 프롬프트를 초안 생성과 실제 파일 생성으로 분리
   - 이유: 전체 교육생이 동시에 파일을 만들면 작업 결과가 제각각 될 수 있다.

8. #33, #34에 선행 조건 명시
   - 이유: ZIP을 먼저 다운로드하고 설치해야 자연스럽다.

9. #41 `.env` 생성 실습을 `.env.example` 우선으로 변경
