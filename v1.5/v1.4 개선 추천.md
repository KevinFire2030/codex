# v1.4 개선 추천

## 배경

- 교육일: 2026-06-15 월요일
- 대상: 지원팀 구성원
- 시간: 2시간
- 진행 방식: 왼쪽에는 `v1.4/index.html` 플레이북, 오른쪽에는 Codex 앱을 띄운 뒤 강사가 실습 내용을 설명하고 교육생은 플레이북의 프롬프트를 복사해 Codex 앱에 붙여 넣어 실행 결과를 확인한다.

현재 v1.4는 초보자용 설명, 보안 원칙, 프롬프트 예시, 예제 파일이 잘 갖춰져 있다. 다만 현장 실습에서는 교육생이 "지금 무엇을 해야 하는지", "정상 결과가 무엇인지", "막히면 어디서 멈춰야 하는지"를 더 빠르게 알 수 있으면 진행 품질이 좋아진다.

## 1. 각 실습마다 "소요 시간 / 완료 기준 / 막히면 확인" 박스 추가

현재 섹션별 설명과 프롬프트는 충분하지만, 교육생 입장에서는 프롬프트 실행 후 어디까지 확인하면 다음 단계로 넘어가도 되는지 헷갈릴 수 있다. 각 주요 실습 앞이나 프롬프트 아래에 작은 체크 박스를 추가하는 것을 추천한다.

예시:

```text
소요 시간: 7분
완료 기준:
- Codex가 프로젝트 목적을 3줄 이상으로 설명했다.
- 주요 폴더와 파일 역할을 구분했다.
- 아직 모르는 부분을 추측이 아니라 "확인 필요"로 표시했다.

막히면 확인:
- Codex 앱에서 선택한 폴더가 v1.4인지 확인한다.
- 프롬프트를 일부만 복사하지 않았는지 확인한다.
```

효과:

- 2시간 교육 중 강사가 매번 "여기까지 되면 됩니다"를 반복 설명하지 않아도 된다.
- 교육생이 자기 화면에서 정상 진행 여부를 판단하기 쉬워진다.
- 실습 속도 차이가 나도 다음 단계 전환이 덜 어수선해진다.

## 2. "초보자용 안전 모드 프롬프트"를 모든 실습의 기본값으로 강조

v1.4는 이미 원본 보존, 민감정보 입력 금지, 읽기 전용 계획 수립을 강조하고 있다. 이를 더 강하게 만들기 위해 각 실습 프롬프트의 첫 문장을 일정한 패턴으로 통일하면 좋다.

추천 문장:

```text
아직 파일을 수정하거나 삭제하지 말고, 먼저 계획과 확인해야 할 점만 알려줘.
```

또는:

```text
원본 파일은 수정하지 말고, 결과가 필요하면 별도 파일로 만들어줘.
```

효과:

- 교육생이 처음부터 Codex에게 과도한 권한을 주는 습관을 줄일 수 있다.
- 파일 정리, 엑셀 병합, 브라우저 자동화처럼 위험도가 다른 실습에도 동일한 안전 기준을 적용할 수 있다.
- 지원팀 실무 적용 시 "계획 먼저, 실행은 승인 후"라는 메시지가 더 잘 남는다.

## 3. 프롬프트 모음에 "난이도 / 사용 상황 / 위험도" 태그 추가

현재 프롬프트 모음은 복사하기 좋게 잘 구성되어 있다. 여기에 태그를 붙이면 교육 후에도 플레이북을 업무 참고자료로 쓰기 쉬워진다.

추천 태그:

- 난이도: 입문, 기본, 응용
- 사용 상황: 폴더 분석, 파일 정리, 메일, 브라우저, 엑셀, 보안
- 위험도: 읽기 전용, 파일 생성, 외부 접속, 승인 필요

예시:

```text
[입문] [폴더 분석] [읽기 전용]
이 폴더가 어떤 프로젝트인지 초보자도 이해할 수 있게 설명해줘.
```

효과:

- 교육생이 "지금 내 업무에는 어떤 프롬프트를 쓰면 되는지" 빠르게 찾을 수 있다.
- 보안상 조심해야 하는 프롬프트를 눈에 띄게 구분할 수 있다.
- 교육 이후 재사용성이 높아진다.

## 4. 실습 전 준비 체크리스트를 첫 화면에 더 크게 배치

교육 방식이 좌우 화면 분할을 전제로 하므로, 첫 화면에서 준비 상태를 바로 확인할 수 있게 하는 것이 좋다. 현재 설명에도 좌우 배치 안내가 있지만, 실습 시작 전 체크리스트로 분리하면 현장 진행이 더 매끄럽다.

추천 체크리스트:

```text
실습 전 준비
- 왼쪽 화면: v1.4/index.html 열기
- 오른쪽 화면: Codex 앱 열기
- Codex 앱에서 프로젝트 폴더 v1.4 선택
- 인터넷 또는 사내망/VPN 연결 상태 확인
- 실제 비밀번호, OTP, 고객정보는 프롬프트에 넣지 않기
- 예제 파일 다운로드 버튼이 동작하는지 확인
```

효과:

- 시작 10분의 환경 세팅 혼선을 줄일 수 있다.
- 강사가 교육생 화면을 빠르게 점검할 수 있다.
- "Codex 앱이 다른 폴더를 보고 있어서 결과가 다르게 나오는 문제"를 줄일 수 있다.

## 5. 강사용 진행 큐와 교육생용 화면 지시를 분리

현재 본문은 설명, 프롬프트, 예시 답변이 한 페이지에 함께 들어 있다. 실습 교육에서는 강사용 멘트와 교육생이 따라 할 액션이 섞이면 화면에서 중요한 지시를 놓칠 수 있다.

추천 방식:

- 교육생용: "복사할 프롬프트", "실행 후 확인할 결과", "다음으로 넘어가기"
- 강사용: "설명 포인트", "주의해서 말할 보안 메시지", "시간이 부족하면 생략할 부분"

예시:

```text
교육생 액션:
1. 아래 프롬프트를 복사한다.
2. Codex 앱에 붙여 넣고 실행한다.
3. 결과에서 주요 폴더 3개 이상이 설명되는지 확인한다.

강사용 큐:
- ChatGPT와 Codex 차이는 "답변자 vs 작업자"로 설명한다.
- 파일 수정은 아직 시키지 않는다고 강조한다.
- 시간이 부족하면 예시 답변 전체 낭독은 생략한다.
```

효과:

- 교육생은 화면에서 자기 행동만 빠르게 찾을 수 있다.
- 강사는 동일한 플레이북을 보면서도 설명 포인트를 놓치지 않는다.
- 이후 다른 강사가 같은 자료로 교육할 때 품질 편차가 줄어든다.

## 우선순위 추천

1. 실습 전 준비 체크리스트 추가
2. 각 실습의 완료 기준 추가
3. 프롬프트 태그 추가
4. 안전 모드 문장 통일
5. 강사용 큐와 교육생 액션 분리

가장 먼저 적용할 것은 1번과 2번이다. 2026-06-15 교육까지 시간이 많지 않다면, 전체 디자인을 크게 바꾸기보다 첫 화면과 각 실습 프롬프트 아래에 체크리스트만 추가해도 현장 진행성이 크게 좋아질 가능성이 높다.
