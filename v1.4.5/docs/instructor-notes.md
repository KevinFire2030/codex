# Instructor Notes v1.4

## 강의 톤

- "AI가 대신 일한다"보다 "작업을 맡기고 검토하는 동료"라는 표현을 사용합니다.
- 개발자가 아닌 참석자도 따라올 수 있도록 폴더, 파일, 변경사항, 승인 같은 단어를 반복해서 설명합니다.
- 사내 시스템 자동화는 가능성보다 권한, 감사 로그, 보안 경계를 먼저 설명합니다.

## 데모 순서

1. Codex 앱에서 `v1.4` 프로젝트 폴더를 선택합니다.
2. 첫 프롬프트로 프로젝트 구조 설명을 요청합니다.
3. 프롬프트 기본기에서 목표, 맥락, 제약, 완료 조건을 보여줍니다.
4. Knox 메일과 브라우징 자동화는 보안 원칙을 먼저 설명한 뒤 시나리오 중심으로 진행합니다.
5. 파일 정리는 "계획만 세우기"에서 시작하고, 실제 이동/삭제는 승인 후에만 수행한다는 점을 강조합니다.
6. 데이터 분석과 엑셀 병합은 원본 보존, 마스킹, 결과 파일 검증을 함께 보여줍니다.
7. `.env`와 DRM 실습에서는 민감정보를 Git에 올리지 않는 원칙을 반복합니다.
8. 마지막에는 프롬프트 모음 섹션에서 실습 프롬프트를 한 번에 복습합니다.

## 막히면 확인할 것

- 프로젝트 폴더가 각자 PC에서 선택한 실습 폴더 기준인지
- Codex가 로컬 실행인지, Cloud/Worktree 실행인지
- Codex가 파일 읽기/쓰기 권한을 가지고 있는지
- 프롬프트에 목표와 완료 조건이 포함되어 있는지
- 민감정보나 DRM 문서 원문을 출력하도록 요청하지 않았는지

## 보안 안내 멘트

사번, 비밀번호, OTP, 토큰, 고객정보, 내부 URL, DRM 문서 원문은 실습 프롬프트에 넣지 않습니다. 실제 업무 자동화는 회사 정책과 시스템 담당자의 승인 범위 안에서만 진행합니다.
